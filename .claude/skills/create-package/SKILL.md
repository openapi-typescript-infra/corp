# Create a TypeScript Package

This skill scaffolds a new TypeScript package in the monorepo under `packages/<category>/<name>/`. Packages are organized by category (auth, cloud, infra, ui).

Ask the user for:
1. **Package name** (e.g. `external-id`, `cloud-sql`)
2. **Category** (e.g. `auth`, `cloud`, `infra`, `ui`)
3. **Description** (one-line)
4. **Dependencies** (if known)

## Directory Structure

```
packages/<category>/<name>/
├── package.json
├── src/
│   └── index.ts
└── .gitignore
```

All other config files (`tsconfig.json`, `tsconfig.build.json`, `eslint.config.mts`, `.prettierrc.yaml`, `vitest.config.ts`, `.commitlintrc.yaml`) are auto-generated by `cpconfig` on `postinstall`. Do NOT create them manually.

## package.json

```json
{
  "name": "@justtellme/<name>",
  "version": "0.0.0",
  "description": "<description>",
  "type": "module",
  "exports": {
    ".": {
      "production": "./dist/index.js",
      "default": "./src/index.ts"
    }
  },
  "main": "./src/index.ts",
  "types": "./src/index.ts",
  "author": "Developers <developers@justtellme.com>",
  "license": "UNLICENSED",
  "scripts": {
    "build": "tsc -p tsconfig.build.json",
    "lint": "eslint .",
    "postinstall": "cpconfig",
    "test": "vitest",
    "typecheck": "tsc --noEmit -p tsconfig.json"
  },
  "keywords": [
    "typescript",
    "justtellme"
  ],
  "config": {
    "cpconfig": "@justtellme/cpconfig"
  },
  "devDependencies": {
    "@justtellme/cpconfig": "workspace:^",
    "@types/node": "^25.5.0",
    "@typescript/native-preview": "^7.0.0-dev.20260314.1",
    "cpconfig": "^1.4.4",
    "eslint": "^10.0.3",
    "prettier": "^3.8.1",
    "typescript": "^5.9.3",
    "vitest": "^4.1.0"
  },
  "dependencies": {}
}
```

Key points:
- `"type": "module"` — all packages are ESM
- Conditional exports: `"production"` points to `dist/`, `"default"` points to `src/` for dev
- `cpconfig` in both `config.cpconfig`, `devDependencies`, and `postinstall` script
- Use `workspace:^` for internal dependencies (e.g. `"@justtellme/external-id": "workspace:^"`)
- If the package has a CLI entry point, add `"bin": "src/cli.ts"`

```

/dist
/*.tsbuildinfo

# Standard
node_modules

coverage
```

## src/index.ts

Create a minimal entry point that exports the package's public API. Keep it simple — just re-exports.

## After Scaffolding

1. The package is automatically part of the workspace because the root `package.json` has `"workspaces": ["packages/*/*"]`
2. Run `yarn` from the monorepo root to install and trigger `cpconfig` (which generates tsconfig, eslint, etc.)
3. The package can be referenced from other packages/services as `@justtellme/<name>` with `"workspace:^"`

## Notes

- Do NOT use `private: true` unless the package should never be published
- The build is plain `tsc` for most packages. If it's a React component library, use `tsup` instead (see ui-kit for that pattern)
- Tests go alongside source files as `*.spec.ts` or `*.test.ts`
