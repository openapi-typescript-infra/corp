# Database Design Conventions

## Migration Tool

- **db-migrate** with `db-migrate-pg` driver
- Config lives in `migrations/db-migrate.json`
- SQL file mode: each migration has a `.js` wrapper that reads from `migrations/sqls/`

## Migration File Naming

- Format: `YYYYMMDDHHmmss-description.js`
- Corresponding SQL files: `sqls/YYYYMMDDHHmmss-description-up.sql` and `-down.sql`
- Descriptions are lowercase, hyphen-separated, imperative: `add-task-idempotency-key`, `seed-task-types`

## Setup Scripts

Located in `migrations/setup/`:

- `db_setup.sql` — creates the database with owner `dbowner`
- `dev_setup.sql` — creates the service user (e.g. `agent-internal`), grants the manager role
- `ci_setup.sql` — creates CI/test users

## Table Naming

- **Plural**, snake_case: `tasks`, `task_steps`, `conversations`, `messages`
- Prefix with domain when tables form a group: `task_types`, `task_steps`, `task_events`

## Column Naming

- snake_case throughout
- Primary key: `{singular_table_name}_id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY`
- UUID column: `{singular_table_name}_uuid uuid NOT NULL DEFAULT gen_random_uuid()`
- Foreign keys: `{referenced_table_singular}_id bigint [NOT NULL] REFERENCES {referenced_table} ({referenced_table_singular}_id)`

## Standard Columns

Every table should include:

```sql
created_at timestamptz NOT NULL DEFAULT CURRENT_TIMESTAMP
updated_at timestamptz NOT NULL DEFAULT CURRENT_TIMESTAMP  -- if rows are mutable
deleted_at timestamptz                                      -- soft delete (nullable)
extra_data jsonb NOT NULL DEFAULT '{}'::jsonb               -- extensibility
```

## Enums

- Custom PostgreSQL enum types: `{name}_enum` suffix
- Example: `CREATE TYPE task_status_enum AS ENUM ('todo', 'in-progress', 'done');`
- Used as column types with defaults: `status task_status_enum NOT NULL DEFAULT 'todo'`

## Indexes

- Name pattern: `idx_{table}_{description}`
- UUID unique index: `CREATE UNIQUE INDEX idx_{table}_by_uuid ON {table} ({table_singular}_uuid);`
- Conditional indexes for soft deletes: `WHERE deleted_at IS NULL`
- Composite indexes for common query patterns: `ON messages (conversation_id, ordinal)`
- Multi-column unique constraints where business logic demands it

## Triggers

- Auto-update `updated_at` on mutable tables:
  ```sql
  CREATE TRIGGER {table_name} BEFORE UPDATE ON {table}
    FOR EACH ROW EXECUTE PROCEDURE set_updated_at();
  ```
- The `set_updated_at()` function sets `NEW.updated_at = now()` and returns `NEW`

## Roles & Grants

- Manager role: `{service}-manager` (e.g. `agent-manager`, `task-manager`)
- Service user: `{service}-internal` (e.g. `agent-internal`)
- Role creation uses `DO $do$...END` blocks with existence checks
- Selective grants: append-only tables get `SELECT, INSERT`; mutable tables get `SELECT, INSERT, UPDATE`
- Always grant sequence usage: `GRANT USAGE ON ALL SEQUENCES IN SCHEMA public TO "{role}";`

## Down Migrations

- Drop tables in reverse creation order
- Drop enums after tables
- Drop functions last
- Always use `IF EXISTS` for idempotent rollbacks

## JSONB Columns

- `extra_data jsonb NOT NULL DEFAULT '{}'::jsonb` on most tables for forward-compatible extensibility
- Use typed jsonb columns for structured data that varies by provider (e.g. LLM message content)
